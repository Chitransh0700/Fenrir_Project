BACKEND SCOPE
═══════════════════════════════════════════

Build a REST API server using Node.js and Express with a MongoDB 
database (or equivalent NoSQL/SQL database of the developer's choice, 
documented in README).

Required API endpoints:

GAMES
GET    /api/games                 list all games, supports query 
                                    params: category, trending, 
                                    featured, search
GET    /api/games/:id             single game detail with screenshots, 
                                    reviews, related games
GET    /api/games/trending         top 6 trending games with rank
GET    /api/games/recommended      recommended for you list

CATEGORIES
GET    /api/categories             list of all categories with live 
                                    game counts

REVIEWS
GET    /api/games/:id/reviews      all reviews for a game
POST   /api/games/:id/reviews      submit a new review (authenticated)

USERS / AUTH
POST   /api/auth/register          create new user account
POST   /api/auth/login             authenticate, return JWT token
GET    /api/auth/me                get current user profile (authenticated)

FAVOURITES
GET    /api/users/favourites       get current user's favourited games 
                                    (authenticated)
POST   /api/users/favourites/:gameId     add game to favourites
DELETE /api/users/favourites/:gameId     remove game from favourites

RECENTLY PLAYED
GET    /api/users/recently-played  get current user's recently played 
                                    games (authenticated)
POST   /api/users/recently-played/:gameId  record a play event

REWARDS / CURRENCY
GET    /api/users/rewards          get current user's coin balance and 
                                    daily reward status (authenticated)
POST   /api/users/rewards/claim    claim daily reward (authenticated)

NEWSLETTER
POST   /api/newsletter/subscribe   subscribe an email to the newsletter

Authentication: JWT-based. Passwords hashed with bcrypt. Protected 
routes require a valid Bearer token.

DATABASE SEED DATA
A seed script must populate the database with:
- Minimum 15 games across the 6 categories, with full metadata 
  matching the structure in the assets document
- Minimum 4 screenshots per game
- Minimum 3 reviews per game
- 6 categories with names and icons
- At least 1 demo user account for testing (credentials documented 
  in README)

TECHNICAL REQUIREMENTS
═══════════════════════════════════════════

Frontend: React.js or Next.js, Tailwind CSS or CSS custom properties
Backend: Node.js + Express, MongoDB (Mongoose) or equivalent
Authentication: JWT
The frontend must fetch all game, category, review, and trending data 
from the backend API  no hardcoded game data in the frontend.
Fully responsive across desktop (1440px), tablet (768px), mobile (375px)
Dark mode is the default and only required theme
All images: placeholder/royalty-free gaming images (picsum.photos, 
unsplash, or similar)  no copyrighted game assets
NSG logo (logo.png) used in navbar, sidebar, and footer exactly as provided
Colour palette from NSG_UI_Design_System.pdf applied throughout
Smooth scroll on horizontal card rows
Hover states on all interactive elements
Environment variables for database connection string and JWT secret, 
documented in .env.example

═══════════════════════════════════════════
OUT OF SCOPE
═══════════════════════════════════════════

The following are explicitly NOT included in this engagement:

- Real multiplayer netcode or a playable game engine (launching a game
  shows a placeholder; no actual gameplay is built)
- Real payment processing or real money in-app purchases (the coin balance
  and daily rewards are simulated only)
- Native iOS or Android applications (delivery is responsive web only at
  1440px, 768px, and 375px)
- Real email delivery for the newsletter (the subscribe endpoint stores the
  email only; no SMTP or transactional email is sent)
- OAuth or social login (authentication is email and password with JWT only)
- Backend features for Community, Friends, Leaderboards, Tournaments, Events,
  Clans, Forums, and Live Streams (these appear as navigation and footer
  links only, with no functional endpoints)
- An admin or content-management dashboard (game data is loaded by the seed
  script only)
- Production hosting, CI/CD, or domain setup (a local end-to-end run is the
  delivery target)
